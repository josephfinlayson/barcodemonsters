Main screen

Animated Monster in centre, eyes moving, keeping user engaged.

As a stretch goal we could integrate the scanned items into this main area of the app. If the user scans an apple, we could take the product image (provided by tesco API) and place in next to the monster

Four touch pages around the monster

  • Top left: scroll through available backgrounds
  • Bottom left: scroll through available backgrounds

  • Top right: work out! - triggers exercise animation
  • Bottom left: clean up! - monster will soil itself from time to time, or throw up if given too many high sugar/high fat foods. Kids love gross stuff!

Monster will tell the user what it needs, it could send the user on treasure hunts, i.e. by asking for an apple. Given that some of our users will be unable to read it might be better to use an icon, or pictures, rather than actually asking for things.

Centre button is the barcode button, triggers barcodescanner plugin

Three tabs on the bottom

  1. Stats: gamify the app, unlock backgrounds/outfits with healthypoints
  2. Main screen: Interact directly with the monster
  3. Basket: look a history of scans, learn more about the foods scanned

Stats screen

Stats screen, an attempt to gamify app

Two graphs

  1. Happiness: goes up if the user engages with the app
  2. Healthiness: matches nutrient profile of scanned items

The user generates 'HealthyPoints', if happiness and healthiness are sustained. Healthypoints could also be generated by succesfully completing a treasure hunt. This allows them to unlock features:

  1. Backgrounds - as you can see the third background is greyed out, with a lock icon
  2. Outfits - in addition, users can scroll along and unlock cooler, and more complex outfits
  3. Accessories - these could also be extra amounts of animations, work-out items
